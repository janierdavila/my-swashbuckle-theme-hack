Getting Started with My API

My API is an HTTP service exposing REST resources over standard HTTP methods. This getting started guide will give you an introduction of what the API supports.

A Word About Swagger

My API exposes a Swagger definition file that you can use to speed up integration. There are lots of tools available, most free, that would allow you to do things like generating clients libraries, test suites, etc. For example, the popular Chrome extension, Postman, allows you to import a Swagger definition file which generates sample requests against all available resources. Our Swagger endpoint is listed in our API Docs. To learn more about Swagger, visit: http://swagger.io

Authentication and Authorization

My API requires clients to authenticate against our Single Sign On (SSO) endpoint. This endpoint requires clients to have valid credentials and a valid client certificate. Our SSO solution uses SAML protocol to exchange claims. A private external document is available with more details about the SAML exchange required to acquire a valid TOKEN. Our SSO endoint is available at https://sso.qpay123.biz/Dap/SignIn (https://sso.qpay123.com/Dap/SignIn in production). Besides issuing a Token, a valid SAML exchange would also provide valid user permissions that clients can use for their authorization needs.

The Token

The SAML protocol lists the Token as a NameID element in the SAML response. The SAML response also contains the Token expiration timestamp. Once you have a valid Token, you can start consuming My API by passing it along with the standard Authorization header, using IAS-TOKEN as the scheme like this: 

Authorization: IAS-TOKEN Hjhiuahskd76876hUIYIUHkjnkuyasldniukulykjnuylijlkhlsdsdd+

Our API Documentation

My API is documented in more details in our API Docs. You can also test our resources by entering a valid token and the required parameters if any. Every resource also documents the type of errors you can expect to receive as well as the meaning of every parameter.
